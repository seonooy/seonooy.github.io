운영체제 스케줄링은 결국 제한된 CPU 시간을 어떤 기준으로 프로세스에 나눠줄지에 대한 문제다.

이번 글에서는 가장 자주 나오는 세 가지를 짧게 다시 정리했다.

## FCFS

먼저 들어온 작업을 먼저 처리하는 가장 단순한 방식이다.

- 구현이 쉽다
- 직관적이다
- 긴 작업이 앞에 있으면 뒤의 짧은 작업들이 오래 기다릴 수 있다

## SJF

실행 시간이 짧은 작업을 먼저 처리하는 방식이다.

- 평균 대기 시간을 줄이는 데 유리하다
- 하지만 실제로는 다음 작업의 실행 시간을 정확히 알기 어렵다

## RR

모든 작업에 같은 시간 조각을 나눠가며 처리하는 방식이다.

- 응답성이 좋아 인터랙티브한 환경에 잘 맞는다
- time quantum이 너무 크면 FCFS처럼 보이고, 너무 작으면 문맥 전환 비용이 커진다

## 메모

개념을 볼 때는 단순히 정의만 외우기보다 "왜 이런 스케줄링이 필요한지"를 같이 보는 편이 이해가 더 잘 됐다.
