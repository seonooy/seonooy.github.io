Docker를 처음 쓸 때는 컨테이너를 띄우는 데만 집중했는데, 여러 서비스가 붙기 시작하면 네트워크 구조를 같이 이해해야 했다.

## 기본 개념

- 컨테이너는 기본적으로 격리된 환경에서 실행된다
- 같은 네트워크에 연결된 컨테이너끼리는 이름으로 통신할 수 있다
- 포트 매핑은 호스트와 컨테이너를 연결하는 개념이다

## 다시 볼 포인트

- `localhost`가 항상 호스트를 의미하는 건 아니다
- 컨테이너 내부에서 보는 주소와 호스트에서 보는 주소가 다를 수 있다
- docker compose를 쓰면 서비스 이름으로 연결하는 흐름이 훨씬 명확해진다
